02 Detection Rules

🌊
Packet Flood
One IP sending a huge burst of packets in a short time — signs of DoS or scanning.
threshold: 100 pkts / 10 sec
🔌
Suspicious Ports
Access to sensitive ports like SSH (22), Telnet (23), RDP (3389) triggers an alert.
ports: 22, 23, 445, 3389 …
🔄
Repeated Connection
Same (src → dst:port) pair hammered repeatedly — brute-force or port scan.
threshold: 50 repeats
🚫
Blacklisted IP
Any packet involving a known-bad IP address triggers an immediate HIGH alert.
source: config.py → BLACKLISTED_IPS

03 Implementation Plan

1
Install dependencies
Install Python 3.10+ and Scapy. On Linux, also install libpcap.
2
Configure rules in config.py
Adjust PACKET_FLOOD_THRESHOLD, TIME_WINDOW, SUSPICIOUS_PORTS, and BLACKLISTED_IPS to match your lab environment.
3
Run unit tests (no root needed)
Verify the detection logic works correctly before touching live traffic.
4
Start the IDS as root/Administrator
Raw packet capture requires elevated privileges on all platforms.
5
Simulate attacks to trigger alerts
Use ping flood, nmap, or a test script to generate detectable traffic and verify alerts appear.
6
Inspect alerts.log
Every alert is timestamped and saved to the log file automatically. Show this in your viva.

04 How to Run & Test

📦 Installation
# Install Scapy
pip install scapy

# Linux: also install libpcap
sudo apt install python3-scapy libpcap-dev
🧪 Run Unit Tests (no root)
python3 test_detector.py

# Expected output:
  ✓ PASS  No alert before 100 packets
  ✓ PASS  Alert fires at 100 packets
  ✓ PASS  Alert fires for port 22 (SSH)
  ...
  5/5 tests passed
🚀 Start the IDS
# Linux / macOS
sudo python3 main.py

# Windows (run terminal as Administrator)
python main.py
💥 Trigger Test Alerts
# Flood alert – ping quickly from another terminal
ping -f 192.168.1.1          # Linux flood ping
ping -t 192.168.1.1          # Windows continuous

# Suspicious port – connect to SSH
ssh user@192.168.1.100

# Repeated connection – quick port scan
nmap -p 22 192.168.1.100

07 Viva Preparation Tips

  • What is an IDS? — A system that monitors network traffic and raises alerts when it detects suspicious patterns. IDS detects (it does not block); an IPS prevents.
  • Why Scapy? — Scapy is a Python library that can capture, parse, craft, and send raw network packets. It wraps libpcap underneath.
  • What is a rule-based IDS? — It compares traffic against predefined rules (thresholds, port lists, IP lists). Fast and explainable, but misses novel attacks.
  • Why does it need root? — Raw socket access (required for packet sniffing below the OS abstraction) is restricted to privileged users on all major OSes.
  • How do you test it without attacking a real host? — Use the unit tests in test_detector.py, or replay a pre-saved PCAP file.
  • What's the difference between TCP and UDP? — TCP is connection-oriented with handshake + reliability; UDP is connectionless and faster but has no delivery guarantee.
  • What is a DDoS? — Distributed Denial-of-Service: many machines flood a target to exhaust its resources. Rule 1 (packet flood) detects single-source variants.
